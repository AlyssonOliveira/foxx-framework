Foxx-Framework is built on ArangoDB database and Foxx services.

Create apps, develop and deploy with ease.

Why?

I wanted to create a tool for building web applications easily using Foxx & ArangoDB. I also wanted to have an easy way to build, develop my app and deploy it with ease. So I did create foxx-framework !
With foxx-framework you can create new skeleton apps with everything configurated. It comes with :

  • brunch for live reload and building assets for production
  • Riotjs for building web apps and widgets.
  • UIKit for designing views.

Get Started.

Install requirements

Foxx-framework needs nodejs and npm installed. ArangoDB of course is needed.

To deploy your application, you'll need Ruby too.

Install foxx-framework

npm install foxx-framework -g

Create your first app

You need first to create an database within ArangoDB web UI.

foxxy new sample_app --database my_test_db

This action will create the skeleton app with a simple login authentification.

cd sample_app && npm install

This will install everything you need to run the application.

Now configure nginx

To make it work properly, we will use Nginx
brew install nginx If needed.

Edit your nginx.conf file and set : 

server {
    listen       8080;
    server_name  sampleapp.dev;

    location / {
        root   /path/to/your/folder/public/;
        index  index.html index.htm;
    }

    location /_db {
            allow all;
            proxy_pass http://127.0.0.1:8529/_db;
    }
}

Notice that in my case Nginx run on port 8080. Open now http://sampleapp.dev:8080/ in your browser.

Setup the application

Open the ArangoDB web UI and go to your database. Open services tab and go to the auth service. Click on the Settings link and set your mailgun credential (Used for sending emails verifications).

Create an account within the app UI and validate it by adding the field { "a": true } to your user object (via ArangoDB UI). So next time you'll login it will be OK.

Run the app

Command

brunch w

This will start the listener ... Any change on your source code will run a live reload in your browser.

Add a CRUD to your app

Command

foxxy g crud post

It will create a Foxx service called posts and will also create a RiotJS widgets called app/widgets/posts.html.tag

Once CRUD created open foxx/posts/APP/main.js at line 30 (line 3 on the code below) ...

... and define your model.

Activating Search functionality

To activate search functionnality you have to define first your search content. Go to line 86 and 96 and replace comment by :

data.search = [data.title, data.text, data.date].join(" ")

So when you'll create or modify your object, the search field will be up to date. You can start using the search bar and press enter to filter your content using the full text search engine.
